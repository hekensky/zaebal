# Журнал трейдера — Telegram-бот

Бот для ведения дневника сделок: тикер, ТВХ, стоп-лосс, тейк-профит, причина входа,
а по факту закрытия — результат, скрин PnL и комментарий.

## Что умеет

- **➕ Новая позиция** — пошагово спрашивает тикер, направление (LONG/SHORT), ТВХ,
  стоп-лосс, тейк-профит и комментарий "почему открываю". Сохраняется как активная.
- **📊 Активные позиции** — список сделок, которые ещё не закрыты. Для каждой можно нажать:
  - ✅ **Закрылась по тейку** → бот просит скрин PnL и комментарий (если ничего не написать
    — подставит одну из "победных" фраз);
  - ❌ **Закрылась по стопу** → бот просит скрин PnL и комментарий, что пошло не так;
  - ⏳ **Не дошло до ТВХ** → бот просит объяснить, почему цена не дошла (без скрина);
  - 🗑 **Удалить** — удалить запись совсем, с подтверждением.
- **📚 История** — все закрытые сделки с пагинацией (по 5 штук).
- **📈 Статистика** — сколько сделок закрыто по тейку/стопу, сколько "не дошло",
  винрейт. В карточке каждой сделки также считается плановое соотношение
  прибыль/риск (R/R), исходя из ТВХ, стопа и тейка на момент открытия.

## Установка и запуск

1. Установи Python 3.10+.
2. Создай виртуальное окружение и поставь зависимости:
   ```bash
   python3 -m venv venv
   source venv/bin/activate   # на Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
3. Получи токен бота у [@BotFather](https://t.me/BotFather) (команда `/newbot`).
4. Скопируй `.env.example` в `.env` и впиши токен:
   ```bash
   cp .env.example .env
   ```
   ```
   BOT_TOKEN=твой_токен_сюда
   ```
5. Запусти бота:
   ```bash
   python bot.py
   ```
6. Напиши боту `/start` в Telegram.

База данных — обычный файл SQLite (`journal.db`), создаётся автоматически рядом с
`bot.py`. Ничего дополнительно поднимать не нужно.

## Куда развивать дальше (идеи)

- Учитывать реальный (не только плановый) результат в % / деньгах, если добавить
  поле "размер позиции" и "цена фактического закрытия".
- Фильтр истории по тикеру или по периоду.
- Экспорт истории в CSV/Excel для анализа.
- Автонапоминание, если позиция "висит" активной дольше N дней.
- Мультипользовательская статистика уже частично готова — все данные привязаны
  к `user_id`, так что бот из коробки работает с несколькими трейдерами
  независимо друг от друга.

## Деплой на сервер (кратко)

Для постоянной работы бота (не только пока запущен у тебя на компьютере) проще
всего запустить его на любом VPS через `systemd` или `screen`/`tmux`, либо
задеплоить как обычное Python-приложение с polling (webhook не обязателен —
в коде используется `start_polling`).
